Kairo Wearable - Hardware Build Guide

Build your own emergency wearable device using an ESP32, OLED display, GPS module, and a few other components. Total cost starts at under 1100 INR.

Assembly Steps

1

Connect OLED to ESP32 via I2C

Wire the SSD1306 OLED display to the ESP32 using the I2C bus. Connect SDA to GPIO21 and SCL to GPIO22. Power the OLED from the ESP32's 3.3V and GND pins. The display uses I2C address 0x3C.

2

Wire SOS Button between GPIO4 and GND

Connect one leg of the momentary push button to GPIO4 and the other leg to GND. The firmware enables the internal pull-up resistor, so no external resistor is needed. When pressed, the pin reads LOW and triggers an SOS alert.

3

Connect GPS Module TX to GPIO16, RX to GPIO17

Wire the NEO-6M GPS module's TX pin to ESP32 GPIO16 (RX2) and the GPS RX pin to ESP32 GPIO17 (TX2). Power the GPS from 3.3V. The module communicates at 9600 baud and will take 30-60 seconds to get a first fix outdoors.

4

Set Up Battery with TP4056 Charge Module

Connect the 3.7V LiPo battery to the TP4056 module: Battery+ to B+ and Battery- to B-. The TP4056 handles safe charging via its micro-USB port and provides overcharge/overdischarge protection.

5

Connect TP4056 Output to ESP32 VIN

Wire TP4056 OUT+ to the ESP32's VIN pin and TP4056 OUT- to GND. The VIN pin has an onboard regulator that accepts 3.7V from the battery and steps it down to 3.3V for the ESP32 and peripherals.

6

Add Voltage Divider for Battery Monitoring

Create a voltage divider using two 100k-ohm resistors in series from Battery+ to GND. Connect the midpoint (junction between the two resistors) to GPIO34. This halves the battery voltage so the ESP32 ADC can safely measure it (max 3.3V input). The firmware maps 3.0V-4.2V to 0-100% charge level.

Firmware Upload Guide

Install Arduino IDE 2.0+ from the official Arduino website. This is the development environment used to compile and upload firmware to the ESP32.
Add ESP32 board support. In Arduino IDE, go to File > Preferences and add this URL to the Additional Board Manager URLs field: https://raw.githubusercontent.com/espressif/arduino-esp32/gh-pages/package_esp32_index.json Then open Tools > Board > Boards Manager, search for esp32, and install the package by Espressif Systems.
Install required libraries. Open Sketch > Include Library > Manage Libraries and search for and install each of these: Adafruit SSD1306, Adafruit GFX, TinyGPSPlus, and ArduinoJson. Accept any dependency prompts that appear.
Open and configure the firmware. Open kairo-device.ino in Arduino IDE. Edit the configuration block at the top of the file: set your WiFi SSID and password, and paste your Kairo's Help profile ID from the dashboard.
Select board and upload. Go to Tools > Board and select ESP32 Dev Module. Select the correct COM port under Tools > Port. Click the Upload button (right arrow icon). The firmware will compile and flash to the ESP32. Open Tools > Serial Monitor at 115200 baud to see debug output.

Power Supply Warning

  • SIM800L requires 3.7V and up to 2A peak current during GSM transmission bursts. The ESP32's onboard regulator cannot supply this. Insufficient power will cause the SIM800L to reset or fail to register on the network.
  • Never power the SIM800L from the ESP32's 3.3V pin. The 3.3V regulator on the ESP32 DevKit can only supply about 500mA, which is far below the SIM800L's peak demand. This can damage both the regulator and the GSM module.
  • Always use the LiPo battery directly for SIM800L VCC. Connect the battery's 3.7V output straight to the SIM800L power input. Add a large capacitor (1000uF electrolytic) across the SIM800L power pins to handle current spikes during transmission.
